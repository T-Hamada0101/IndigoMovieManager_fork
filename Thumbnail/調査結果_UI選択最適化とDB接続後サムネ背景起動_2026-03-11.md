# 調査結果: UI選択最適化とDB接続後サムネ背景起動

更新日: 2026-03-11
対象: `MainWindow.Search.cs` `MainWindow.xaml.cs` `Thumbnail/MainWindow.ThumbnailCreation.cs` `UserControls/ExtDetail.xaml.cs` `UserControls/TagControl.xaml.cs`

## 変更要点
- 検索・ソート・タグ絞り込み後の `SelectFirstItem()` 呼び出しを外した。
  - 画面側が勝手に先頭選択へ飛ばないようにする。
- 一覧更新を「表示中タブだけ」に絞る補助メソッドを追加した。
  - `ApplyFilterListToTabIndex`
  - `RefreshItemsControlByTabIndex`
  - `UpdateExtDetailFromCurrentSelection`
- サムネイル background worker / coordinator / progress viewer の起動を、DB有効化後へ寄せた。
  - `EnsureThumbnailBackgroundServicesForCurrentDb`
  - `TryGetCurrentMainDbPathForThumbnailBackgroundWork`

## 期待効果
- 検索やタグ操作のたびに意図しない先頭選択へ戻る挙動を避ける。
- 非表示タブの不要な再描画を減らす。
- DB未接続やDB切替途中で worker 系を起こしてしまう無駄な起動を避ける。
- 詳細ペインは無選択時に閉じ、選択中だけ表示する。

## 確認結果
- `Debug|x64` のソリューションビルド成功。
- この束は `DropTool/Worker` 調査や workthree 文書群とは独立して切り出し可能。
