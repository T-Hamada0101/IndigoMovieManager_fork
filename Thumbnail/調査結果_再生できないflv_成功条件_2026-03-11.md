# 再生できない.flv 成功条件メモ

## 1. 対象

- 対象動画: `E:\_サムネイル作成困難動画\flv\再生できない.flv`
- 目的: retained log から「サムネイル取得成功時の条件」を固定し、同条件で再現確認できるようにする

## 2. 結論

- retained log 上では、この動画は 2026-03-11 だけでなく、2026-03-06 以降に複数回 `autogen success` が出ている
- したがって「今回が履歴上の初成功」ではなく、「いま観測した成功条件を固定できた」が正確
- 成功時の本線条件は、今のところ `repair` や `ffmpeg1pass fallback` ではなく、`autogen` 単独成功である

## 3. ログで確認できた成功条件

### 3.1 movieinfo probe

`debug-runtime.log` では 2026-03-10 21:19:52 に以下を確認できた。

- `ffmediatoolkit success=True duration_sec=478.95 fps=19.938 frames=9549 codec='flv1'`
- `opencv success=True duration_sec=478.947 fps=19.938 frames=9549`
- `autogen success=True duration_sec=478.95 fps=19.938 frames=9549 codec='flv1' detail='format_duration'`

ここから分かること:

- コンテナ情報の取得自体は壊れていない
- `autogen` でも動画長・fps・frame数は取れている
- 少なくとも retained log 上では、成功の前提条件として `codec=flv1` と `detail='format_duration'` が揃っている

### 3.2 実サムネイル成功

`thumbnail-create-process.csv` では以下を確認できた。

- `2026-03-11 02:33:43.976 再生できない.flv autogen success`
- `2026-03-11 02:41:23.164 再生できない.flv autogen success`

さらに worker log では以下を確認できた。

- `2026-03-11 02:33:43.990 thumbnail done role=Normal movie='E:\_サムネイル作成困難動画\flv\再生できない.flv' output='...\Thumb\111\160x120x1x1\再生できない.#ef6c367c.jpg'`
- `2026-03-11 02:28:40.388 thumbnail done role=Normal movie='E:\_サムネイル作成困難動画\flv\再生できない.flv' output='...\Thumb\111\120x90x3x1\再生できない.#ef6c367c.jpg'`
- `2026-03-11 02:28:40.396 thumbnail done role=Normal movie='E:\_サムネイル作成困難動画\flv\再生できない.flv' output='...\Thumb\111\160x120x1x1\再生できない.#ef6c367c.jpg'`
- `2026-03-11 02:41:23.175 thumbnail done role=Normal movie='E:\_サムネイル作成困難動画\flv\再生できない.flv' output='...\Thumb\111\120x90x1x1\再生できない.#ef6c367c.jpg'`

ここから分かること:

- 成功ロールは `Normal`
- 出力は `Thumb\111\...` 配下
- 1枚構成でも 3枚構成でも成功履歴がある
- retained log 上では `repair start` / `repair success` / `engine fallback` がこの動画に対して紐づいておらず、成功は `autogen` 直通と見るのが妥当

## 4. 今回の再現条件として固定する値

次回の再現確認では、最低限以下が揃うことを成功条件とする。

1. 対象動画が `E:\_サムネイル作成困難動画\flv\再生できない.flv`
2. `movieinfo-probe` で `autogen success=True`
3. `movieinfo-probe` で `duration_sec=478.95` 前後、`fps=19.938`、`frames=9549`、`codec='flv1'`
4. `thumbnail-create-process.csv` に `movie_file_name='再生できない.flv' engine=autogen status=success`
5. worker log に `thumbnail done role=Normal movie='...再生できない.flv'`

## 5. 再現手順

### 5.1 手動再現

1. PowerShell で repo 直下へ移動する
2. 次を実行する

```powershell
pwsh -NoLogo -File .\Thumbnail\Test\run_autogen_e2e_manual.ps1 -Engine autogen
```

3. アプリで、対象動画を含む DB を開く
4. `再生できない.flv` のサムネイル生成を実行する
5. 1枚系を先に確認する
   - `160x120x1x1`
   - `120x90x1x1`
6. 余力があれば 3枚系も確認する
   - `120x90x3x1`
7. 完了後、保存されたログで以下を確認する
   - `thumbnail-create-process.csv`: `autogen success`
   - worker log: `thumbnail done role=Normal`

### 5.2 判定ポイント

- 成功:
  - `thumbnail-create-process.csv` に `再生できない.flv autogen success`
  - worker log に `Thumb\111\...` への出力完了
- 条件不一致:
  - `movieinfo-probe` で duration / fps / frames がズレる
  - `autogen success=True` が出ない
- 別経路:
  - `repair` や `ffmpeg1pass fallback` が出た場合は、今回固定した成功条件とは別扱いにする

## 6. 補足

- 同時刻帯では `NGカット.tmp.flv` が `Specified method is not supported.` で失敗している行もある
- そのため、今回の観測は「FLV 全般が直った」ではなく、「`再生できない.flv` は現行条件で `autogen` 単独成功が再現可能」と整理する
- 今後コードへ落とす時は、まず `movieinfo-probe` の成立条件と `autogen` 直通成功の因果を優先して切り分ける
