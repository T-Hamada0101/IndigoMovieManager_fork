# 優先順位表 サムネイルスレッド再構築計画向け 難読動画論点 2026-03-11

## 1. 目的
- `サムネイルスレッド再構築計画` で、難読動画由来の論点をどの順で設計へ反映すべきかを固定する。
- 個別動画の再調査順ではなく、レーン設計と救済責務の反映優先度を示す。

## 2. 優先順位表

| 優先 | 対象 | 現在の結論 | 反映先 | 扱い |
| --- | --- | --- | --- | --- |
| P1 | `35967型` | 長尺 `No frames decoded` だが `ffmpeg midpoint success` | `retry policy` | すぐ反映 |
| P1 | `画像1枚あり顔` 型 | 超短尺で中央失敗、極小 seek のみ成功 | `ffmpeg1pass` 短尺 fallback | すぐ反映 |
| P1 | `repair/remux 先頭救出` 型 | `インデックス破壊-093-2-4K` のように先頭有効区間だけ救える | `repair workflow` | すぐ反映 |
| P2 | `真空エラー2` 型 | `autogen demux immediate eof`、`ffmpeg1pass success` | `retry policy` | 反映済み確認を維持 |
| P2 | FLV1 成功群 | `movieinfo-probe` 成立時は `autogen` 直通成功 | 通常系維持 | 設計原則へ反映 |
| P2 | `古い.wmv` 型 | 成功優勢だが `No frames decoded` / `Specified method is not supported.` に揺れる | 通常系維持 | 設計原則へ反映 |
| P3 | true near-black 群 | `near-black` 固定、`No frames decoded` とは別系統 | `autogen` frame 選定 | 比較継続 |
| P3 | `画像1枚ありページ.mkv` | 顔型に誤適用できない | 誤適用防止 | 比較継続 |
| P3 | `【ライブ配信】神回scale_2x_prob-3.mp4` | 超短尺で末尾だけ明るい | 別エージェントで engine 側修正 | 委譲 |

## 3. 優先順の意味

### P1
- レーン設計へ直接返せる。
- `通常系で引きずらない`
- `ゆっくり系へ渡す条件が明確`
- `動画名ベタ判定不要`

### P2
- 再構築計画の設計原則に効く。
- すでに一部本体反映済み、または「形式だけで振り分けない」根拠として使う。

### P3
- 今は設計へ固定しない。
- 比較継続、誤適用防止、または別エージェント委譲が妥当。

## 4. 計画書へ先に入れるべき文
- 通常系は初回 `autogen` を担当し、失敗後の重い救済は保持しない。
- ゆっくり系は `retry policy` によって選ばれた長尺 no-frames 群、超短尺 tiny-seek 群、repair/remux 群を担当する。
- repair は全編復旧だけでなく、先頭有効区間だけでもサムネイル取得可能な状態へ縮退させる責務を持つ。
- FLV1 や WMV3 のような古い形式でも、成功実績がある限り形式だけでゆっくり系固定にしない。
