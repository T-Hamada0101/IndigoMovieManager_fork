# 連絡用 サムネイルスレッド再構築計画向け 別ツリー成果要約 2026-03-11

## 1. 目的
- 別ツリーで蓄積した難読動画の成果 doc を、`サムネイルスレッド再構築計画` に流し込みやすい形へ圧縮する。
- 個別動画の詳細ではなく、「通常系で持つべき責務」「ゆっくり系へ送るべき条件」「まだ本線設計へ返さない論点」を整理する。
- `要件定義_サムネイルスレッド制御とレーン設計_2026-03-06.md` と `仕様書_サムネイル並列方式再設計_2026-03-08.md` の判断材料にする。

## 2. 先に結論
- 再構築計画へ先に返すべき成果は 3 系統でよい。
  1. `35967型`
  2. `画像1枚あり顔` 型
  3. `repair/remux で先頭有効区間だけ救える途中破損群`
- 逆に、すぐには再構築計画へ返さず、比較継続または別エージェント対応に回すべきものも明確になった。
  - true near-black 固定群
  - `画像1枚ありページ.mkv`
  - `【ライブ配信】神回scale_2x_prob-3.mp4`
  - `古い.wmv` のような成功優勢だが揺れる動画

## 3. レーン設計へ返すべきこと

### 3.1 通常系の責務
- 初回は原則 `autogen` を先に試す。
- 拡張子や codec だけでゆっくり系へ送らない。
- `movieinfo-probe` が成立する個体、FLV1 のように実績上 `autogen` 成功する個体は通常系に残す。
- `古い.wmv` のような「成功優勢だが揺れる」個体も、即ゆっくり系固定にはしない。

### 3.2 ゆっくり系へ送るべき代表条件
- `35967型`
  - `autogen / service = No frames decoded`
  - 長尺
  - `ffmpeg midpoint success`
  - 意味: 通常系で引きずらず、ゆっくり系 `ffmpeg1pass` 候補へ落とす代表条件
- `画像1枚あり顔` 型
  - `autogen / service = No frames decoded`
  - 超短尺
  - `0.001 / 0.01` 秒の極小 seek だけ成功
  - 意味: 通常系に重い再試行を残さず、短尺 fallback をゆっくり系へ分離する代表条件
- 途中破損 remux 群
  - 例: `インデックス破壊-093-2-4K.mp4`
  - 元動画は中間以降が壊れている
  - `remux` 後に読める先頭有効区間だけ救出でき、その短縮版なら `autogen / ffmpeg1pass` が通る
  - 意味: 「repair は全編復旧だけではない」。ゆっくり系 repair/remux が先頭有効区間だけでも救出できる

### 3.3 通常系とゆっくり系を混ぜない方がよい論点
- true near-black
  - `No frames decoded` とは別系統
  - フレーム選定、黒判定、採用条件の問題
- `画像1枚ありページ.mkv`
  - `画像1枚あり顔` と似るが同条件では救えない
  - 短尺だから一括救済、を防ぐ比較役
- `【ライブ配信】神回scale_2x_prob-3.mp4`
  - `duration <= 0.2s`
  - 末尾だけ明るい
  - 短尺 fallback 候補が早すぎて踏めていない
  - これはスレッド再構築より、engine 側の短尺 fallback 設計論点

## 4. 動画群ごとの再構築計画への意味

| 対象群 | 現状の要点 | 再構築計画への意味 | 差し込み先 |
| --- | --- | --- | --- |
| `35967型` | `autogen` 失敗、`ffmpeg midpoint` 成功、`ffmpeg1pass` 成功 | 通常系で粘らず、ゆっくり系 one-pass へ渡す基準 | `retry policy` |
| `画像1枚あり顔` 型 | 超短尺、中央 seek 失敗、極小 seek のみ成功 | 超短尺救済を通常系から分離する基準 | `ffmpeg1pass` 短尺 fallback |
| `インデックス破壊-093-2-4K` 型 | 元動画は破損、remux 後短縮版で成功 | ゆっくり系 repair/remux の価値を裏づける | `repair workflow` |
| FLV1 成功群 | `movieinfo-probe` が成立すれば `autogen` 直通成功あり | codec ベタ判定禁止の根拠 | 通常系維持 |
| `真空エラー2` 型 | `autogen` は demux 即 EOF、`ffmpeg1pass` 成功 | 初回失敗後の one-pass 救済へ直結できる | `retry policy` |
| `古い.wmv` 型 | 成功優勢、たまに揺れる | 形式だけでゆっくり系固定しない比較役 | 通常系維持 |
| true near-black | `near-black` 固定 | スレッド設計より frame 選定設計 | 今は保留 |
| `画像1枚ありページ` 型 | 顔型に誤適用できない | 誤適用防止の比較役 | 今は保留 |
| `【ライブ配信】神回...` 型 | 超短尺、終盤だけ明るい | スレッド設計より短尺 fallback 候補設計 | 別エージェント対応 |

## 5. 再構築計画で採用したい原則
- 初回 `autogen` は通常系で処理する。
- 初回失敗後に重い救済を通常系へ居座らせない。
- ゆっくり系へ送る条件は、動画名ではなく `decision_basis` で表現する。
- `No frames decoded` は 1 種類ではないので、少なくとも次を分ける。
  - 長尺 + midpoint success
  - 超短尺 + tiny-seek success
  - demux immediate eof + onepass success
  - true near-black ではない超短尺終盤明フレーム
- `repair` は「全編復旧」だけでなく、「読める先頭有効区間だけ救出してサムネ取得」を目的にしてよい。

## 6. 再構築計画へ今すぐ入れてよいもの
- `35967型` の `retry policy` 反映
- `画像1枚あり顔` 型の短尺 fallback 反映
- repair/remux をゆっくり系責務として明示すること
- FLV1 や WMV3 を形式だけでゆっくり系へ落とさない原則

## 7. まだ入れないもの
- bitrate 閾値の決め打ち
- 動画名ベタ判定
- `FailureKind` の追加前提設計
- true near-black の救済条件
- `画像1枚ありページ.mkv` の扱い確定
- `【ライブ配信】神回scale_2x_prob-3.mp4` 向けの短尺終盤明フレーム救済

## 8. 再構築計画書へ追記したい文言の素案
- 通常系は初回 `autogen` を担当し、失敗後の重い救済は保持しない。
- ゆっくり系は巨大動画、再試行動画、repair/remux、`ffmpeg1pass` 救済を担当する。
- ゆっくり系への移送条件は `decision_basis` で管理し、拡張子や動画名では固定しない。
- `repair` は全編復旧だけでなく、読める区間だけでもサムネイル取得可能な状態へ縮退させる責務を持つ。
