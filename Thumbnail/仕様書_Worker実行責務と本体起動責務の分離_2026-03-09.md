# 仕様書 Worker実行責務と本体起動責務の分離 2026-03-09

## 1. 目的

- Worker 実行中の挙動を Worker 側へ集約する。
- 本体側は Worker 起動前の初期条件だけを持つ。
- GPU 使用設定と Windows 優先度設定の責務境界を明確にする。

## 2. 結論

責務は下記で分離する。

### 2.1 Worker 側

- 並列数調整
- 今のプリセット
- GPU 使用の実効値
- in-process エンジンスレッド優先度
- 子プロセス起動時の優先度反映ルール

### 2.2 本体側

- `worker.exe` の起動時のみのプロセス優先度
- GPU 使用の希望値を Worker 起動前に渡すだけ

## 3. 用語定義

### 3.1 GPU 使用の希望値

- 本体側が Worker 起動前に渡す入力値。
- 例: `RequestedGpuEnabled`
- 本体側はこの値を「入力として送るだけ」であり、実行中の実効値の管理主体ではない。

### 3.2 GPU 使用の実効値

- Worker 側が実際に採用している値。
- 例: `EffectiveGpuEnabled`
- Worker 側は環境、プリセット、エンジン都合、失敗時の代替経路を含めて最終値を決める。

### 3.3 Worker プロセス優先度

- `worker.exe` 自体の Windows プロセス優先度。
- 本体側が起動時に指定する。
- 実行中の変更責務は持たない。

### 3.4 in-process エンジンスレッド優先度

- Worker プロセス内で動く C# 実行スレッドの優先度。
- Worker 側が保持し、実行時に適用する。
- `Thread.Priority` や同等の内部制御が対象である。

### 3.5 子プロセス起動時の優先度反映ルール

- `ffmpeg.exe` など外部プロセス起動時に、どの Windows 優先度で開始するかのルール。
- Worker 側が保持し、起動のたびに適用する。

## 4. 境界ルール

### 4.1 本体側の責務

- Worker 起動前に初期条件を渡す。
- Worker 起動後の細かい実効制御はしない。
- Worker 実行中の設定整合性は Worker 側の責務とする。

### 4.2 Worker 側の責務

- 実行中に参照される設定は Worker 側が正とする。
- 並列数、プリセット、GPU 実効値、スレッド優先度、子プロセス優先度ルールは Worker 側で完結させる。
- 本体から受け取った希望値は、Worker 側で実効値へ解決してから使う。

## 5. 優先度仕様

### 5.1 `worker.exe` の優先度

- 本体側が起動時に設定する。
- 例: `BelowNormal` / `Idle`
- 起動後に本体が追従変更しない。

### 5.2 in-process エンジンスレッド優先度

- Worker 側でのみ管理する。
- 同一 Worker プロセス内で動く処理にだけ効く。
- 外部起動した `ffmpeg.exe` には直接効かない。

### 5.3 子プロセス優先度

- `ffmpeg.exe` のような外部プロセスは、子プロセス起動時の優先度反映ルールで制御する。
- `in-process エンジンスレッド優先度` と混同しない。

## 6. GPU 使用仕様

### 6.1 本体側

- `GPU 使用の希望値` を Worker 起動前に渡す。
- 本体側はこの値の最終採用可否を持たない。

### 6.2 Worker 側

- 受け取った希望値を初期入力とする。
- エンジン要件、失敗時の代替経路、環境条件を踏まえて `GPU 使用の実効値` を決める。
- 実際に使った値は Worker 側の状態として保持する。

## 7. 破綻防止ルール

- `GPU 使用` は二重管理しない。
- 本体側は `RequestedGpuEnabled`、Worker 側は `EffectiveGpuEnabled` として別名・別責務で扱う。
- `worker.exe` 優先度と `in-process エンジンスレッド優先度` は別設定として扱う。
- `in-process エンジンスレッド優先度` は外部プロセスに効く前提で設計しない。

## 8. 推奨データ構造

### 8.1 本体 -> Worker 起動入力

- `WorkerProcessPriority`
- `RequestedGpuEnabled`

### 8.2 Worker 実効設定

- `MaxParallelism`
- `Preset`
- `EffectiveGpuEnabled`
- `InProcessEngineThreadPriority`
- `ChildProcessPriorityRule`

## 9. 非対象

- 本体側から Worker 実行中の `worker.exe` 優先度を随時変更する機能
- 本体側から Worker 実効値を直接上書きする機能
- `Thread.Priority` を外部プロセスへ直接伝播させる設計

## 10. 実装時の注意

- 本体側 UI は「起動条件の入力」に留める。
- Worker 側 UI / Coordinator 側 UI は「実効状態の表示と運転」に寄せる。
- 設定名は `希望値` と `実効値` を必ず分ける。
- ログでも `requested` と `effective` を分けて出す。
