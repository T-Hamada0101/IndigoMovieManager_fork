# AI向け 難読動画調査再現ガイド 2026-03-11

## 1. 目的
- 難読動画の調査を、別の AI や別ツリーでも同じ精度で再現できるように手順化する。
- 「勘で触る」のではなく、観測ログと実画像で原因を切り分ける。
- 実装判断に使える形で、原因、再現条件、成功条件、未解決点を残す。

## 2. この調査の基本原則
- 1 本の動画に対して、`ffprobe`、アプリ内部ログ、実画像の 3 系統を必ず突き合わせる。
- `ffprobe` だけで原因を断定しない。
- `autogen` 単体の挙動と、アプリ本線の worker 経路の挙動を分けて観測する。
- 「失敗した」では終えず、どの時刻で、どの engine で、何が返ったかまで固定する。
- 成功時は必ず画像を動画フォルダへ出力し、目視確認まで行う。
- 再現性のため、実行時のビルド状態、ログ時刻、入力パス、engine 条件を文書へ残す。

## 5. 標準手順

### 5-1. 素性確認
- まず `ffprobe` でコンテナの表面情報を取る。
- 見る値:
  - `duration`
  - `nb_frames`
  - `avg_frame_rate`
  - `codec_name`
  - `pix_fmt`
  - `bit_rate`
- ここでは「仮説」を置くだけで、断定しない。

例:
```powershell
ffprobe -v error -select_streams v:0 -show_entries stream=codec_name,pix_fmt,avg_frame_rate,nb_frames,width,height -show_entries format=duration,bit_rate,size -of default=noprint_wrappers=1 "E:\path\movie.mp4"
```

### 5-2. `autogen` 単体ベンチ
- まず `autogen` 単体で動かし、1 本の動画に対する最小観測を取る。
- 成功したら `-ExportToMovieFolder` を付けて、実画像を動画と同じフォルダに出す。

例:
```powershell
pwsh -File .\Thumbnail\Test\run_thumbnail_engine_bench.ps1 `
  -InputMovie 'E:\path\movie.mp4' `
  -Engines autogen `
  -Iteration 1 `
  -Warmup 0 `
  -Priority Normal `
  -Configuration Debug `
  -Platform x64 `
  -ExportToMovieFolder
```

### 5-3. `ffmpeg1pass` と比較
- `autogen` が失敗したら、同条件で `ffmpeg1pass` を流す。
- `autogen` 失敗でも `ffmpeg1pass` 成功なら、動画全体が壊れているのではなく、`autogen` 経路の問題か、one-pass 救済向き動画の可能性が高い。

例:
```powershell
pwsh -File .\Thumbnail\Test\run_thumbnail_engine_bench.ps1 `
  -InputMovie 'E:\path\movie.mp4' `
  -Engines ffmpeg1pass `
  -Iteration 1 `
  -Warmup 0 `
  -Priority Normal `
  -Configuration Debug `
  -Platform x64 `
  -ExportToMovieFolder
```

### 5-4. 本線ログの確認
- ベンチ結果だけで終えず、`thumbnail-create-process.csv` と `debug-runtime.log` を同時に見る。
- 確認項目:
  - `success / failed`
  - `error_message`
  - fallback の有無
  - `movieinfo-probe`
  - `engine fallback`
  - seek 系 debug ログ

例:
```powershell
Get-Content 'C:\Users\na6ce\AppData\Local\IndigoMovieManager_fork\logs\thumbnail-create-process.csv' -Encoding UTF8 | Select-String 'movie.mp4'
```

```powershell
Get-Content 'C:\Users\na6ce\AppData\Local\IndigoMovieManager_fork\logs\debug-runtime.log' -Encoding UTF8 | Select-String 'movie.mp4|autogen-|engine fallback|movieinfo-probe'
```

### 5-5. 実画像の目視確認
- `success` でもログだけでは不十分。
- 動画フォルダへ出した JPEG を開き、黒画像、壊れ画像、76 bytes の擬似出力でないことを確認する。
- `output_bytes` も bench CSV で確認する。

## 6. 判定の観点

### 6-1. `probe固定` の意味
- この調査でいう `probe固定` は `ffprobe` 固定ではない。
- 正しくは、成功時の `movieinfo-probe` 値を比較基準として固定すること。
- 主に固定する値:
  - `duration_sec`
  - `fps`
  - `frames`
  - `codec`
  - `detail`

### 6-2. 失敗分類
- `No frames decoded`
  - 超短尺 1 フレーム動画
  - demux 即 EOF
  - 長尺の途中破損
  - seek 後 packet 0 件
- `Autogen produced a near-black thumbnail`
  - 要求秒より手前の暗フレームを拾っている可能性
- `ffmpeg one-pass failed`
  - 動画自体の破損、または one-pass 条件不適合
- `Specified method is not supported.`
  - 特殊データやデコーダ経路差の可能性

### 6-3. 代表的な成功パターン
- FLV1 でも `movieinfo-probe` が実用値を返すなら `autogen` 直通成功する個体がある。
- 超短尺 1 枚動画は `EOF` 後 drain で救える。
- 短尺先頭黒動画は、要求秒より前の frame を捨てると `autogen` で救える。
- `autogen demux immediate eof` 群は、`ffmpeg1pass` 救済が有効なことがある。
- 長尺 low-bitrate / partial file 群は、`autogen` 本線改善より repair / remux / onepass 側で扱う方が筋が良い。

## 7. 切り分けのコツ
- `duration` に対して `size` が極端に小さい場合は partial file を疑う。
- `8:20 まで再生可` のような実再生観察は、ログより強い証拠になることがある。
- `ffprobe` は読めるが中間時刻抽出が壊れる場合、インデックス不整合ではなく途中欠損のことがある。
- `success` ログがあっても、出力 JPEG が実体かを必ず確認する。
- 同じタイトル断片の別ファイル名が混ざる場合は、完全一致と部分一致を分けて調べる。

## 8. 調査結果として必ず残す項目
- 対象動画の完全パス
- 実行日時
- 実行 engine
- 成功 / 失敗
- 失敗メッセージ
- `ffprobe` の主要値
- `movieinfo-probe` の主要値
- 実際に出た画像パス
- bench CSV のファイル名
- runtime log の該当キーワード
- 原因仮説
- 次に試すべき 1 手

## 9. 実務上の禁止事項
- `ffprobe` の結果だけで「動画破損」と断定しない。
- ベンチの `success` だけを見て「取得できた」と断定しない。
- 古いビルドのまま調査を続けない。
- 別動画の成功履歴を同一動画の成功履歴として扱わない。
- 実ログを見ずに「fallback が動かなかった」と断定しない。
