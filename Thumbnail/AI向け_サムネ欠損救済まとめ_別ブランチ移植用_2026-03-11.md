# AI向け_サムネ欠損救済まとめ_別ブランチ移植用_2026-03-11

## 1. このまとめの目的
- 別ブランチへ `サムネ欠損救済` を移植する時に、AI が「どの塊を」「どの順で」持っていくかを短時間で判断できるようにする。
- 今回の救済は大きく 2 系統ある。
  - `Watch / Manual` 起点で、欠損した jpg を再投入する救済
  - `autogen` 起点で、短尺難読動画から 1 枚を拾う救済

## 3. 移植対象の要点

### 3.1 Watch / Manual 側の欠損サムネ救済
- 目的
  - `動画更新なし + jpg削除` で Watch 差分が拾えないケースを、低頻度の再走査で救う。
- 実装の核
  - `Watcher/MainWindow.Watcher.cs`
  - DB + タブ単位で rescue scope を作る。
  - 同一 scope は `60秒` の最小間隔を掛ける。
  - Watch 自動実行は高負荷時に見送るが、Manual は見送らない。
  - 候補は DB を 1 回走査してメモリバッファへ積む。
  - 既存 jpg または `ERRORマーカー` があるものは候補へ入れない。
  - 実投入はキュー空き枠ぶんだけ段階投入する。
  - quota が途中で減っても、未投入分は先頭へ戻して次回へ持ち越す。
  - Watch 起動直後だけ、自動救済を 1 回見送って初回表示と競合させない。

### 3.2 ERRORマーカーとの共存
- 目的
  - 「最終失敗として固定した動画」を自動救済で無限再投入しない。
  - 手動再作成 / 手動救済では stale な失敗固定マーカーを外して再実行できるようにする。
- 実装の核
  - `Watcher/MainWindow.Watcher.cs`
    - `ShouldSkipThumbnailEnqueueBecauseMarkerExists(...)`
    - `ShouldSkipThumbnailEnqueueBecauseMarkerExistsForTab(...)`
  - `Thumbnail/MainWindow.ThumbnailCreation.cs`
    - 手動作成と救済再投入の前で `TryDeleteThumbnailErrorMarker(...)` を呼ぶ

### 3.3 autogen 側の短尺欠損救済
- 目的
  - 短尺・少パネルの難読動画で、黒コマ即採用や seek ずれによる欠損を減らす。
- 実装の核
  - `Thumbnail/Engines/FfmpegAutoGenThumbnailGenerationEngine.cs`
  - 短尺・少パネル時だけ `ShouldPreferClosestNonBlackThenLatestBright(...)` を有効化する。
  - `要求秒に整合する非黒フレーム` を最優先にする。
  - 近傍が黒しかない時だけ `latest bright` を最後の救済候補にする。
  - 通常動画へは広げない。

## 4. AIが移植時に守るべき単位
- 最小単位A
  - `Watcher/MainWindow.Watcher.cs` の rescue scope / throttle / buffer / drain 一式
- 最小単位B
  - `Watcher/MainWindow.Watcher.cs` の ERRORマーカー判定一式
  - `Thumbnail/MainWindow.ThumbnailCreation.cs` の marker 削除
- 最小単位C
  - `Thumbnail/Engines/FfmpegAutoGenThumbnailGenerationEngine.cs` の短尺救済
  - `Tests/IndigoMovieManager_fork.Tests/FfmpegAutoGenThumbnailGenerationEngineTests.cs`

## 5. 推奨移植順
1. `Watcher/MainWindow.Watcher.cs` の rescue 制御と buffer を移す。
2. `ERRORマーカー` 判定と手動再投入前の marker 削除を移す。
3. `MissingThumbnailRescuePolicyTests.cs` を移して、救済ポリシーだけ先に固定する。
4. `FfmpegAutoGenThumbnailGenerationEngine.cs` の短尺救済を移す。
5. `FfmpegAutoGenThumbnailGenerationEngineTests.cs` を移して、短尺判定と選択ルールを固定する。

## 6. 移植後の確認ポイント
- Watch 自動実行
  - 高負荷時は rescue を見送ること
  - 起動直後の 1 回目だけ startup-skip すること
  - 同一 DB + タブで 60 秒以内に連打しないこと
- 候補構築
  - 既存 jpg がある動画を拾わないこと
  - `ERRORマーカー` がある動画を拾わないこと
  - quota が途中で減っても候補を失わないこと
- 手動再投入
  - marker を外してから再投入すること
- autogen
  - 短尺・少パネル時だけ近傍の非黒優先 + latest bright fallback が効くこと
  - 通常動画では従来の代表フレーム選択を崩さないこと

## 7. 補足
- この救済は「全部を毎回再スキャンする」設計ではない。
- Watch 自動実行は通常キュー優先、Manual は救済優先、という非対称ルールが肝。
- `ERRORマーカー` 共存を落とすと、自動救済が失敗固定動画を再投入し続けて挙動が崩れる。
