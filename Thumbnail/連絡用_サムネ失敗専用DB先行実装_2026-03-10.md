# 連絡用: サムネ失敗専用DB 先行実装依頼（2026-03-10）

## 1. 目的
- サムネ失敗専用DBを、別AIエージェント側で先に実装してもらうための連絡用ドキュメント。
- こちら側は後続で `サムネ失敗` タブ表示、`FailureKind` 連携、`HangSuspected` 表示へ接続する。
- 先行実装側では、専用DBの schema / service / insert 経路を先に固めてほしい。

## 2. 依頼したい実装範囲
- サムネ失敗専用DB の schema 定義
- サムネ失敗専用DB の path 解決
- 読み書き service
- DebugMode 時の失敗 insert API
- 現在 MainDB 単位での分離
- 最低限の取得 API

今回の先行実装では、UI 変更は必須ではない。

## 3. この実装の前提
- `サムネ失敗` タブは将来 QueueDB 直読ではなく、サムネ失敗専用DBを正として表示する。
- DebugMode ではサムネ失敗の全件を専用DBへ insert する。
- 通常運用時に常時 insert するかどうかは後で切り替え可能にしたい。
- MainDB 本体 (`*.wb`) の schema は変更しない。
- QueueDB もこのタスクでは壊さない。

## 4. 必須要件

### 4.1 DB の単位
- MainDB ごとに分離すること
- `MainDbFullPath` に対応付くこと
- path 解決は QueueDB と同様に、`MainDbFullPath` 正規化 + hash 系でよい

### 4.2 最低限保存する列
- `DbName`
- `MainDbFullPath`
- `MoviePath`
- `PanelType`
- `MovieSizeBytes`
- `Duration`
- `Reason`
- `FailureKind`
- `AttemptCount`
- `OccurredAtUtc`
- `UpdatedAtUtc`

### 4.3 追加してほしい列
- `TabIndex`
- `OwnerInstanceId`
- `WorkerRole`
- `EngineId`
- `QueueStatus`
- `LeaseUntilUtc`
- `StartedAtUtc`
- `LastError`
- `ExtraJson`

### 4.4 取得API
- 現在 MainDB 向けの全件取得
- `OccurredAtUtc DESC` の新しい順
- 将来フィルタしやすい DTO で返すこと

### 4.5 insert API
- DebugMode 時に失敗1件を追記できること
- upsert よりまずは append 前提でよい
- 途中失敗も最終失敗も記録できること

## 5. 後続側が必要としている契約
- 専用DB path を `MainDbFullPath` から一意に解決できること
- `GetFailureRecords(mainDbFullPath)` 相当があること
- `InsertFailureRecord(record)` 相当があること
- `FailureKind` を文字列ではなく enum または固定値として保持しやすいこと
- `ExtraJson` などで将来項目を逃がせること

## 6. 推奨実装形
- 新規候補:
  - `src/IndigoMovieManager.Thumbnail.Queue/FailureDb/ThumbnailFailureDebugDbPathResolver.cs`
  - `src/IndigoMovieManager.Thumbnail.Queue/FailureDb/ThumbnailFailureDebugDbSchema.cs`
  - `src/IndigoMovieManager.Thumbnail.Queue/FailureDb/ThumbnailFailureDebugDbService.cs`
  - `src/IndigoMovieManager.Thumbnail.Queue/FailureDb/ThumbnailFailureRecord.cs`

命名は多少変わってもよいが、`FailureDb` 系でまとまっていると後続接続がしやすい。

## 7. insert 呼び出し候補
- `ThumbnailQueueProcessor`
- failure finalizer
- retry 戻し分岐
- `HangSuspected` 分岐
- placeholder 失敗分岐

ただし先行実装では、まず service と schema だけでもよい。

## 8. 今回はまだ不要なもの

- DataGrid 列追加
- `FailureKind` 絞り込み UI
- 運転席への表示
- QueueDB との完全同期
