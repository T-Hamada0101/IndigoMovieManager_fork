# Implementation Plan UIシンプル化 設定フォーム左ペイン化 2026-06-25

最終更新日: 2026-06-27

## 目的

左ドロワーを廃止し、メイン画面上は「新規作成」「ファイルを開く」「設定」の三ボタンだけを残す。ドロワーに残っていた設定・ツール・最近使った管理ファイルの導線は、設定フォーム内で「左がカテゴリ、右が詳細」の形へ寄せる。

## 実装方針

- メイン画面の左ドロワーとハンバーガーボタンは廃止する。
- メイン画面上に残す入口は、新規作成、ファイルを開く、設定の三ボタンだけにする。
- 共通設定画面を `TabStripPlacement="Left"` のカテゴリ型フォームへ変更する。
- 共通設定画面の最小幅は、左カテゴリと固定幅入力欄が横方向に詰まらない幅で固定する。
- 最近使った管理ファイルと表示件数は、共通設定画面の「管理ファイル」カテゴリへ移す。
- 現在DBの個別設定は、共通設定画面の「現在DB」カテゴリへ集約する。
- 監視フォルダ編集、監視フォルダ更新チェック、全ファイルサムネイル再作成は、共通設定画面の「ツール」カテゴリから既存の MainWindow 処理へ渡す。
- DB保存は既存の `PersistDbSettingsValues(...)` を使い、保存経路を増やさない。

## 禁止線

- 左ドロワー、`MenuToggleButton`、`MenuConfig` / `MenuTool` のツリーを戻さない。
- 最近使った管理ファイルをメイン画面のドロワー前提 UI へ戻さない。
- 設定画面から直接 DB 更新ロジックを複製しない。
- 設定フォームのカテゴリ切替で重いDB読込や監視処理を即時実行しない。

## 検証

- `CommonSettingsWindow.xaml` が左カテゴリ/右詳細の構成を持つこと。
- `CommonSettingsWindow.xaml` の最小幅が、左カテゴリ/右詳細の固定幅行を隠さないこと。
- `MainWindow.xaml` に `DrawerHost` / `MenuToggleButton` / `MenuConfig` / `MenuTool` が残っていないこと。
- `MainWindow.xaml` には `BtnNew` / `BtnOpen` / `BtnSettings` の三ボタンだけが管理ファイル入口として残っていること。
- 「管理ファイル」カテゴリから最近使った管理ファイルを既存の MainWindow DB 切替処理へ渡すこと。
- ツールカテゴリのボタンは MainWindow の既存処理へ橋渡しするだけであること。
