# UsnMft機能概要（2026-03-06）

## 1. 目的
- `Watcher/UsnMftProvider.cs` が何をしているかを、運用者と実装者の両方が短時間で把握できるようにする。
- `everything` 代替としての役割、現在の実動作、制約を明文化する。

## 2. 結論
- 設定キー `usnmft` を選ぶと、`UsnMftProvider` が `IFileIndexProvider` 契約で動作する。
- 現在のアプリ実装は **Windows限定** で、`Lite.FileIndexService` を使って対象ルートを走査する。
- `usnmft` は `BackendMode = AdminUsnMft` 固定、`StandardFileSystem` は別プロバイダへ分離済み。
- `usnmft` は管理者権限が必要で、非管理者環境では `availability_error:AdminRequired` を返して通常監視へフォールバックする。
- ルート単位のインデックスキャッシュを持ち、短時間の連続呼び出しでは再構築を抑制する。

## 3. 呼び出し経路
1. `MainWindow.Watcher` が `FileIndexProviderFactory.CreateFacade()` で Facade を生成する。
2. `FileIndexProviderFactory` が設定値を正規化し、`everything` 以外を `usnmft` として扱う。
3. `usnmft` の場合は `UsnMftProvider` を生成する。
4. `IndexProviderFacade` 経由で以下を実行する。
   - 動画候補収集: `CollectMoviePaths`
   - サムネイル既存確認: `CollectThumbnailBodies`

## 4. 動画候補収集の流れ
`CollectMoviePaths(FileIndexQueryOptions options)` は次の順で動く。

1. 引数と `RootPath` を検証する。
2. `CheckAvailability()` で Windows かどうかを確認する。
3. ルートパスを正規化する。
   - 末尾スラッシュあり
   - 末尾スラッシュなし
4. `CheckExt` から対象拡張子集合を作る。
   - 例: `*.mp4,*.mkv` -> `.mp4`, `.mkv`
5. `ChangedSinceUtc` を UTC に正規化する。
6. ルート単位キャッシュから `Lite.FileIndexService` を取得する。
7. 必要時のみ `RebuildIndexAsync()` を実行する。
8. `Search("", 1000000)` で全件近い一覧を取得し、ホスト側で絞り込む。
9. 以下の条件で対象動画だけ残す。
   - ディレクトリではない
   - 指定ルート配下である
   - `IncludeSubdirectories=false` 時は直下のみ
   - 対象拡張子に一致する
   - `ChangedSinceUtc` 以降に更新されている
10. `MoviePaths`、最大更新日時、reason文字列を返す。

## 5. キャッシュ仕様
- キー: ルートパス単位
- クールダウン: 3秒
  - 前回インデックス構築から3秒未満なら再構築を省略し、`index=cached` を返す。
- 追いつき条件:
  - `ChangedSinceUtc` があり、保持中インデックス時刻がそれより古い場合は再構築する。
- アイドル破棄: 20分未使用で破棄
- 最大保持数: 32ルート
  - 上限超過時は最近使っていないルートから破棄する。
- テスト補助API:
  - `GetCacheEntryCountForTesting()`
  - `GetCacheCapacityForTesting()`
  - `ClearCacheForTesting()`

## 6. サムネイル確認の流れ
`CollectThumbnailBodies(string thumbFolder)` は `thumbFolder` 直下の `*.jpg` を列挙し、ファイル名からボディ部を抽出する。

- `alpha.#abcd.jpg` -> `alpha`
- `beta.jpg` -> `beta`

用途は「既にサムネイルが存在する動画本体名の集合」を高速に作ること。

## 7. reason文字列
成功時は `ok:` 系を返し、ホスト側が Prefix で解釈する。

代表例:
- `ok:provider=usnmft index=rebuilt indexed_at=... count=12`
- `ok:provider=usnmft index=cached indexed_at=... count=12 since=...`
- `ok`

失敗時は既存Everything互換の reason 契約へ寄せる。
- 動画候補収集失敗: `everything_query_error:{ExceptionType}`
- サムネイル確認失敗: `everything_thumb_query_error:{ExceptionType}`
- 非Windows: `everything_not_available`

## 8. 現在の実装上の注意
- `UsnMftProvider.CheckAvailability()` は Windows 判定のみで、管理者権限や USN 利用可否までは見ていない。
- `CreateService()` では `StandardUserExcludePaths = []` を渡しているため、内包ライブラリの既定除外（`.git`, `node_modules`）は使わない。
- 実際の絞り込みは `Search()` の後でホスト側が行うため、ルート内件数が多いほどメモリと走査コストが増える。
- `SearchLimit` は `1_000_000` 固定。
- `CollectThumbnailBodies()` はサブフォルダを見ない。

## 9. 内包ライブラリとの関係
- 内包先: `src/IndigoMovieManager.FileIndex.UsnMft`
- `FileIndexService` 自体は `Auto / AdminUsnMft / StandardFileSystem` を持つ。
- 現在のホスト側の分担は次の通り。
  - `EverythingProvider`: Everything IPC
  - `UsnMftProvider`: `AdminUsnMft`
  - `StandardFileSystemProvider`: `StandardFileSystem`
- そのため、3つの実装をUIから明示的に切り替えられる状態になっている。
