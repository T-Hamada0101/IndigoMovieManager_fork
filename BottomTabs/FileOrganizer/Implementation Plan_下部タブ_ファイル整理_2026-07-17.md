# Implementation Plan 下部タブ「ファイル整理」 2026-07-17

最終更新日: 2026-07-18

変更概要:

- メイン一覧をクリックするとAvalonDock上の整理タブが非active扱いになり選択同期を捨てていたため、詳細同期をDock活性から分離した。ショートカットの明示的な有効条件は登録行のONだけに一本化した。
- タブを「移動対象動画の詳細」と「登録リスト」の左右2領域へ分割した。
- 詳細は代表1件のサムネイル、動画名、ファイルパス、フォルダパスを表示し、各コピーとフォルダを開く操作を追加した。
- 登録先ごとのON、現在表示中の全動画を対象にする「全移動」、全ボタンのツールチップを追加した。
- ショートカットと全移動の実行前に、代表ファイル名と件数を示す確認ダイアログを追加した。
- 移動先登録時、現在DBの監視対象外なら監視フォルダへ追加するか確認し、承認時は既存の監視登録経路へ接続するようにした。
- 移動先表示用の読み取り専用プロパティを `TextBox.Text` へ明示的な `OneWay` で結び、初回タブ描画時の双方向バインド例外を防止した。

## 目的

- メインタブで選択した動画を、あらかじめ登録した移動先へ素早く移動する。
- 対象行がONの時だけ `Ctrl+1` から `Ctrl+9` を有効にし、誤操作を防ぐ。
- WhiteBrowser の `.wb` スキーマを増やさず、既存の `movie_path` 更新契約を再利用する。

## 実装方針

- 下部へ `ToolFileOrganizer` を常設し、古い `layout.xml` では既存の必須タブ復元と同じ導線で救済する。
- 移動先9件は `Properties.Settings.FileOrganizerDestinations` に保存する。
- 物理ファイル移動は既存の `QueueMovieFileMove(...)` を使い、UIスレッドを塞がない。
- 物理移動に成功したレコードだけ、既存の背景保存で `.wb` の `movie_path` を更新する。
- ショートカット表示は、対象行がONの時だけアクセント色にする。ONは安全のため起動時にOFFへ戻す。
- 未登録、移動先消失、未選択は移動せず、タブ下部の状態表示で理由を伝える。
- 通常ショートカットは左の詳細表示中1件だけを対象にする。
- 「全移動」は `FilteredMovieRecs` の現在表示中全件を対象にし、詳細表示は代表1件と件数だけにする。
- 移動前は必ず代表ファイル名、件数、移動先を確認する。
- 監視範囲は同一フォルダまたは `watch=1 / sub=1` の親フォルダ配下を対象とし、範囲外の登録先だけ監視追加を確認する。

## 受け入れ条件

- `Ctrl+1` から `Ctrl+9` とテンキーの同キーで、詳細表示中の1件を登録先へ移動できる。
- 対象行がOFFの時は反応しない。メイン一覧へフォーカスを移してもONは有効なままにする。
- 「全移動」で現在表示中の全動画を登録先へ移動できる。
- 詳細に代表サムネイル、ファイルパス、フォルダパスと各操作ボタンを表示できる。
- すべての操作ボタンに説明ツールチップがある。
- 通常移動と全移動の前に代表ファイル名と件数を確認できる。
- 監視対象外の移動先を登録した時だけ、監視フォルダ追加を選択できる。
- 移動先登録は次回起動後も残る。
- `.wb` のスキーマ変更と一覧全件再読込を増やさない。
- 保存済みの古いドックレイアウトでもファイル整理タブが表示される。
