# AI向け 大機能詳細理解書（全体インデックス）

最終更新日: 2026-03-11

## 1. 目的

この資料は、AIエージェントがコードベースを短時間で把握し、改修時の影響範囲を外さないための「大機能単位の入口」です。  
詳細は下記の個別理解書を参照してください。

## 1.5 このブランチで先に読む資料

このブランチでは、大機能の理解より先に、以下のブランチ方針を必ず読むこと。

- `AI向け_ブランチ方針_サムネ高速感優先_2026-03-11.md`

特にサムネイル、Watcher、Queue、UI反映タイミングに触る場合は、この方針の判断軸を個別設計より優先すること。
当面の目標は 2 つあり、優先順位は以下で固定する。

1. UI の詰まり解消
2. 難読動画のサムネイル全作成

ただし 2 は、表示・キュー・サムネ作成のテンポを悪化させないことが前提である。

## 2. 大機能マップ

- 01: 起動・DB切替・UI初期化
  - `Docs/AI向け_詳細理解書_01_起動DB切替UI初期化_2026-03-07.md`
- 02: 監視・ファイルサーチ・MainDB登録
  - `Docs/AI向け_詳細理解書_02_監視ファイルサーチMainDB登録_2026-03-07.md`
- 03: サムネイル作成管理（投入/永続化/消費）
  - `Docs/AI向け_詳細理解書_03_サムネイル作成管理_2026-03-07.md`
- 04: 検索・履歴・画面絞り込み
  - `Docs/AI向け_詳細理解書_04_検索履歴画面絞り込み_2026-03-07.md`

## 3. 先に読む順序（推奨）

1. 01で「MainWindow中心の責務境界」とDB切替時の安全設計を把握  
2. 02で「新規動画検出〜DB登録〜キュー投入」の前段パイプラインを把握  
3. 03で「QueueDB中心の後段パイプライン（作成管理）」を把握  
4. 04で検索UXと履歴DB更新の挙動を把握

## 4. 改修時の最低チェックリスト

- DB切替中の処理混入ガードを壊していないか
- 0KB動画の除外とエラーマーカー運用を壊していないか
- QueueDBの重複圧縮（MainDB + MoviePathKey + TabIndex）を壊していないか
- Searchイベントでサムネイル再起動が必要な箇所を見落としていないか


## 5. 更新時の記録ルール

- 対象資料の「最終更新日」を必ず更新する
- どの大機能に対する変更か（01〜04 / フロー資料 / 構成書）を明記する
- 変更理由を1〜3行で追記し、コード変更と同じPRで管理する
- ブランチ方針に関わる変更では、当面の目標 2 つへの影響も明記する


## 6. PR作成時の最小記載ルール

- 変更した資料名（ファイルパス）を必ず列挙する
- どの大機能に関する更新か（01〜04 / フロー資料 / 構成書）を本文に明示する
- 変更理由・影響範囲・確認コマンドを簡潔に記載する
